2Ô∏è‚É£ What sudo vppctl Actually Does

vppctl is NOT VPP.

It is just the CLI client tool.

Architecture:
+----------------+
|     vppctl     |  <-- CLI client
+----------------+
        |
        |  (socket connection)
        v
+----------------+
|      VPP       |  <-- Main dataplane process
+----------------+

So when you do:

sudo vppctl

It connects to:

/run/vpp/cli.sock

But this only works if:

VPP is already running in background

Usually started via systemd service:

sudo systemctl start vpp

So:

Command	What it does
vpp	Starts VPP process
vppctl	Connects to running VPP
